# Aster Code v0.1.0 — First Local Release

**Date:** July 5, 2026
**Build:** `npm run release:local`
**Platform:** Windows 10/11 (x64), NSIS installer
**Repository:** `mertg/aster-code`

---

## What Works ✅

### Agent Loop
- **Agent plan generation** — Classifies tasks into 7 types (explain, plan, edit-code, debug-build, ui-fix, dependency-fix, docs)
- **Skill routing** — 14 intent categories matched to 8 built-in skills with confidence scores and risk levels
- **Approval-gated execution** — All plans require user approval before execution; no autonomous file edits or commands
- **Session management** — Create agent sessions, generate plans, approve/reject, view execution activity

### Workbench
- **File tree explorer** — Browse workspace files with create/delete, file-type icons
- **Multi-tab editor** — Open multiple files, save, unsaved indicators
- **Terminal panel** — Click-to-run commands (npm install, npm run dev), logs display
- **Live preview** — Browser-frame iframe preview of running dev server

### Desktop App
- **Electron shell** — Native Windows window (1440×900, ivory theme)
- **Runtime auto-start** — Spawns runtime server as child process on launch
- **Health monitoring** — Pings `localhost:3001/health` every 5 seconds
- **Secure IPC** — Runtime status, restart, and logs via preload bridge
- **NSIS installer** — Desktop/Start Menu shortcuts, customizable install directory

### Model Registry
- **5 provider adapters** — Ollama, LM Studio, OpenRouter, NVIDIA NIM, OpenAI-compatible
- **Mock model data** — Deterministic model metadata for all providers
- **Provider config** — UI toggles, URL fields, API key inputs in Settings
- **Cache status** — TTL-based model cache with manual/auto-refresh

### Settings
- **Provider configuration** — Toggle providers, set endpoints and API keys
- **System prompt library** — Full CRUD with tags, export/import JSON, localStorage persistence
- **Runtime management** — Status display, restart button, log viewer (desktop only)
- **OAuth scaffolding** — GitHub/Google login buttons (disabled, not implemented)

### MCP Gateway
- **Server registry** — 4 default server configs (Filesystem, GitHub, Postman, Memory)
- **Tool discovery** — Mock tool generation with risk categories
- **Policy engine** — 5-layer access control, blocked tools, approval requirements
- **Audit logging** — In-memory audit trail of all invocations

### First-Run Experience
- **Welcome banner** — Onboarding card with "What Works Now" and "Coming Next" panels
- **Suggested prompts** — One-click prompts ("explain this project", etc.)
- **Disabled feature indicators** — Clear badges for LLM, OAuth, MCP, external skills status

## Known Limitations

### Packaging
- **No code signing** — Expect SmartScreen "Run anyway" prompt; no EV/OV cert yet
- **Default Electron icon** — Custom Aster Code icon not yet created (`assets/icon.ico` is missing)
- **~343 MB installer** — Runtime `node_modules` adds significant size (~50MB); future bundling optimization planned
- **Windows only** — No macOS/Linux installer targets configured

### Runtime
- **No `.env` in package** — Providers must be configured via Settings UI after install
- **In-memory sessions** — Agent sessions lost on runtime restart
- **No persistent storage** — Workspace files are temp-only, no user data persistence

### Features
- **No streaming** — Agent execution returns single response, no SSE streaming yet
- **No auto-update** — Manual reinstall required for new versions
- **36 npm vulnerabilities** — Audit fix recommended before production use
